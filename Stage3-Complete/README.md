## How to Use Specialists

### If You Have MCP Installed

Simply address the specialist by name:

```
Dean, review this file for performance issues
```

```
Roger, check this file for code quality problems
```

```
Uma, review the user experience
```

```
Jordan, analyze integration and extensibility
```

The specialist automatically retrieves BC best practices and applies them.

### If You Don't Have MCP

No problem! Reference the `.al-guidelines` folder:

```
Review knowledge files in .al-guidelines/domains/dean-debug/ then analyze this file for performance issues
```

Both approaches work - this way just relies on the agent to understand how to apply the knowledge.

## What Makes Specialists Different?

**Generic AI** (Stage 2):
- General programming knowledge
- Common patterns
- No platform expertise

**BC Specialists** (Stage 3):
- Deep BC platform knowledge
- Specific BC patterns (SIFT keys, permission sets, etc.)
- Production-ready recommendations

## The Big Picture

Software development isn't just coding. It includes:
- Performance
- Security
- User experience
- Extensibility
- Error handling
- Telemetry
- Documentation

**Developers are strongest at coding.** Time pressure means we often skip these other dimensions.

**BC Specialists cover all dimensions** - catching issues you don't have time to find manually.

This is **augmentation**, not replacement. AI helps you be production-ready faster.
